1.1.4

LAN

LAN or Local Area Network is a type of network that consists of a small amount of receiving devices, or hosts. Hosts are also sometimes called end devices. A LAN can range in size from small (2 hosts) to large (hundreds of hosts)

SOHO

SOHO or Small Office / Home Office is a type of LAN that is specifically for small offices, home businesses, or remote work. All SOHOs are LANs, but not all LANs are SOHOs

For example, to speak, humans use the same technology, vocal cords. However, if we do not use our vocal cords to make certain sounds when talking, we cannot understand each other. This is where languages come in. Languages use certain sounds paired with one another to form words and thoughts. If one of these sounds is not used correctly, it becomes harder to understand. Communication Protocols are like languages. There are many languages, but they all can convey the same message, but it will sound, or transmit, differently Protocols govern communication between end devices Protocols define the method, language, and confirmation of data being received To do this, you need a defined sender and receiver, a common protocol Network devices follow similar rules to human communication Devices must be identified by a MAC or IP address Devices must use a common protocol such as TCP-IP or Ethernet Devices synchronize when messages are sent, I.E one after another in a modern WLAN Devices send ACK packets to acknowledge that they received data Examples of Protocols HTTP/HTTPS is used in web communication SMTP is an email protocol FTP is the file transfer protocol TCP is used in the transport layer for reliable delivery IP - Internet Protocol Protocol Characteristics Message Format Messages must use a common format. They are converted into this format using encoding. The type of format is determined by the channel and medium that the message is being transmitted on Message Size The message size is critical. The amount of data transmittable is 46-1500 bytes in each frame/packet These can be different depending on the protocol Segmentation Messages are segmented into smaller parts if a message is too large. This process is called segmentation Timing Timing is essential in networking to avoid collisions. Timing is the synchronization and coordination of sending and receiving messages CSMA/CD Carrier Sense Multiple Access / Collision Detection Used in legacy networks. This only used in half-duplex legacy networks, and CSMA/CA is used in half-duplex technologies today When each device sends data, it might collide in the medium, and then cause a ‘collision’. CSMA/CD detects these collisions and tells the device to send the data again CSMA/CA Carrier Sense Multiple Access / Collision Avoidance Used in half-duplex wireless networks today When each device sends data, it doesn't allow any other device to send data until the device is finished sending Half/Full - Duplex Whether or not a device can send and receive data at the same time Legacy ethernet devices could only operate in half-duplex Nowadays all ethernet devices are full-duplex All wifi devices are half-duplex (or so we’ve been told) Encoding Turning a message into a format that can be transmitted and received correctly Encapsulation The process of adding control information to data, to transmit it correctly For example, in the network layer, the IP header and trailer is added. Test Telnet from the PC On PC-A: Go to Desktop → Command Prompt Type: telnet 192.168.1.2 (Replace 192.168.1.2 with your switch’s IP.) 3. What you should see If it works: You’ll see something like: Trying 192.168.1.2 ... Open User Access Verification Password: After you enter the VTY password, you should get the Switch> prompt. If it doesn’t work: “Connection refused” or “timed out” → check: IP address / subnet Default gateway (if different network) line vty config transport input includes telnet Commands DO NOT ENTER THE COMMANDS STARTING WITH CAPITAL LETTERS IT WILL BREAK THE PACKET TRACER TO ESCAPE IP DOMAIN LOOKUP, USE CTRL+SHIFT+6 START IN (config) MODE ip domain-name abcd.com username admin secret password crypto key generate rsa general-keys modulus 1024 ip ssh version 2 line vty 1 3 transport input ssh login local logging synchronous exit Test connectivity with the command ‘ssh -l [username] [ip address]’ from any PC connected to the router You can replace ssh with telnet in the command ‘transport input’ I do not know how to do ssh/telnet, we are not doing this today (thankfully)
